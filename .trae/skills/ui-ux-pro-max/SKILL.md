# UI/UX Pro Max - 设计智能

跨平台UI/UX设计指南，包含67种UI样式、161个调色板、57种字体配对、161种产品类型推理规则、99条UX指南和25种图表类型。支持15+技术栈（React、Next.js、Vue、Svelte、SwiftUI、React Native、Flutter、Tailwind、shadcn/ui等）。

## 何时使用

此技能应在以下情况下使用：

### 必须使用

- 设计新页面（落地页、仪表盘、管理后台、SaaS、移动应用）
- 创建或重构UI组件（按钮、模态框、表单、表格、图表等）
- 选择配色方案、字体系统、间距标准或布局系统
- 审查UI代码的用户体验、可访问性或视觉一致性
- 实现导航结构、动画或响应式行为
- 做出产品级设计决策（风格、信息层次、品牌表达）
- 改善界面的感知质量、清晰度或可用性

### 推荐使用

- UI看起来"不够专业"但原因不明
- 收到关于可用性或体验的反馈
- 发布前UI质量优化
- 跨平台设计对齐（Web / iOS / Android）
- 构建设计系统或可复用组件库

### 跳过

- 纯后端逻辑开发
- 仅涉及API或数据库设计
- 与界面无关的性能优化
- 基础设施或DevOps工作
- 非视觉脚本或自动化任务

**决策标准**：如果任务会改变功能的**外观、感觉、动效或交互方式**，应使用此技能。

## 使用方法

### 步骤1：分析用户需求

从用户请求中提取关键信息：
- **产品类型**：娱乐（社交、视频、音乐、游戏）、工具（扫描器、编辑器）、生产力（任务管理、笔记）
- **目标受众**：C端消费者用户；考虑年龄段、使用场景
- **风格关键词**：活泼、活力、极简、暗色模式、内容优先、沉浸式等
- **技术栈**：React、Vue、Next.js、Flutter等

### 步骤2：生成设计系统（必需）

**始终从`--design-system`开始**获取带有推理的完整推荐：

```bash
python3 .trae/skills/ui-ux-pro-max/scripts/search.py "<产品类型> <行业> <关键词>" --design-system [-p "项目名称"]
```

此命令会：
1. 并行搜索多个域（产品、风格、颜色、落地页、字体）
2. 应用`ui-reasoning.csv`中的推理规则选择最佳匹配
3. 返回完整设计系统：模式、风格、颜色、字体、效果
4. 包含需要避免的反模式

**示例：**
```bash
python3 .trae/skills/ui-ux-pro-max/scripts/search.py "美容 spa 健康 服务" --design-system -p "宁静水疗"
```

### 步骤3：补充详细搜索（按需）

获取设计系统后，使用域搜索获取更多细节：

```bash
python3 .trae/skills/ui-ux-pro-max/scripts/search.py "<关键词>" --domain <域> [-n <最大结果数>]
```

**可用域：**

| 域 | 用途 | 示例关键词 |
|----|------|-----------|
| `product` | 产品类型推荐 | SaaS、电商、作品集、医疗、美容 |
| `style` | UI样式、颜色、效果 | 玻璃拟态、极简主义、暗色模式 |
| `typography` | 字体配对、Google字体 | 优雅、活泼、专业、现代 |
| `color` | 按产品类型的调色板 | saas、电商、医疗、美容、金融 |
| `landing` | 页面结构、CTA策略 | hero、证言、定价、社交证明 |
| `chart` | 图表类型、库推荐 | 趋势、比较、时间线、漏斗 |
| `ux` | 最佳实践、反模式 | 动画、可访问性、z-index、加载 |
| `google-fonts` | 单个Google字体查询 | sans serif、monospace、variable |

### 步骤4：技术栈指南

获取特定技术栈的实现最佳实践：

```bash
python3 .trae/skills/ui-ux-pro-max/scripts/search.py "<关键词>" --stack <技术栈>
```

**可用技术栈：** react、nextjs、vue、nuxtjs、svelte、swiftui、react-native、flutter、html-tailwind、shadcn、angular、laravel

## 常见UI专业规则

### 图标与视觉元素

| 规则 | 标准 | 避免 |
|------|------|------|
| 不用emoji作为图标 | 使用SVG图标（Lucide、Heroicons） | 使用emoji（🎨 🚀 ⚙️）作为导航或系统控件 |
| 触控目标最小值 | 最小44×44pt（iOS）或48×48dp（Android） | 小图标无扩展点击区域 |
| 图标一致性 | 使用一个图标集/视觉语言 | 同一层级混用填充和描边图标 |

### 交互（应用）

| 规则 | 正确做法 | 错误做法 |
|------|----------|----------|
| 点击反馈 | 80-150ms内提供清晰的按下反馈 | 点击无视觉响应 |
| 动画时长 | 微交互150-300ms，使用平台原生缓动 | 瞬时过渡或慢动画(>500ms) |
| 禁用状态清晰度 | 使用禁用语义、降低强调、无点击动作 | 看起来可点击但无响应的控件 |

### 明暗模式对比度

| 规则 | 正确做法 | 错误做法 |
|------|----------|----------|
| 文本对比度（亮色） | 正文文本对比度≥4.5:1 | 低对比度灰色正文 |
| 文本对比度（暗色） | 主文本对比度≥4.5:1，次文本≥3:1 | 暗色模式下文本与背景融合 |
| 令牌驱动主题 | 使用语义化颜色令牌 | 硬编码每屏十六进制值 |

### 布局与间距

| 规则 | 正确做法 | 错误做法 |
|------|----------|----------|
| 安全区域合规 | 尊顶部/底部安全区域 | 将固定UI放在刘海或手势区域下 |
| 8dp间距节奏 | 使用一致的4/8dp间距系统 | 随机间距增量无节奏 |
| 可读文本宽度 | 大设备上保持长文本可读 | 全宽长文本影响可读性 |

## 发布前检查清单

在交付UI代码前，验证以下项目：

### 视觉质量
- [ ] 不使用emoji作为图标（使用SVG）
- [ ] 所有图标来自一致的图标系列和风格
- [ ] 按下状态视觉效果不会导致布局偏移
- [ ] 语义化主题令牌使用一致

### 交互
- [ ] 所有可点击元素提供清晰的按下反馈
- [ ] 触控目标满足最小尺寸要求
- [ ] 微交互时长保持在150-300ms范围
- [ ] 禁用状态视觉清晰且不可交互

### 明暗模式
- [ ] 主文本对比度在两种模式下≥4.5:1
- [ ] 分隔线/边框和交互状态在两种模式下可区分
- [ ] 模态/抽屉遮罩透明度足够强

### 布局
- [ ] 安全区域得到尊重
- [ ] 滚动内容不被固定/粘性栏遮挡
- [ ] 在小手机、大手机和平板上验证

### 可访问性
- [ ] 所有有意义的图像/图标有可访问性标签
- [ ] 表单字段有标签、提示和清晰的错误消息
- [ ] 颜色不是唯一指示器
- [ ] 支持减少动效和动态文本大小
